# E3 ‚Äî Component-library linking (publish / link / instance / re-derive)

Status: **DONE 2026-07-15.** `cargo test --workspace` green (new: sync-core
`binfile::trim_to_single_file` + `lock::` `LibraryLink`/`plan_relink` tests).
`cargo clippy` clean on the touched crates. The bounded E3 gate

Chapter 3's linking half (PLAN3.md): a component-library `.penpot` published as
a **shared library** (`set-file-shared`), a consumer file **linked** to it
(`link-file-to-library`) that places a component **instance** referencing the
library by its **vault-local file-id**, and a **surgical linked-file export**
that keeps that reference on disk *without* inlining the library. The disposable
`file_library_rel` is re-derived from the per-vault lockfile on a delete-DB
rebuild (invariant 1); library change severity is surfaced through the **E1
contract-diff channel**, never `revn`.

## What E3 builds on (reuse, not reinvent)

- **The live probe is the seed.** `scripts/ecosystem-spike/e3_probe.py` proved
  the three never-before-run library RPCs and the on-disk caveat against Penpot
  2.16.2. `scripts/e3-library.sh` promotes that exact RPC sequence to a
  house-style PASS/FAIL gate driving the real package verbs + the daemon's
  automatic M2 resurrect and boot re-link reconcile.
- **Installer + lockfile = E2.** Publish/link route through the E2 install path
  (`packages::do_install`, `installer::` import-as-new + settle-to-fixpoint) and
  the E2 `lock.json`. The forward-compatible `#[serde(default)] links` field on
  `LockEntry` was reserved by E2 exactly for this ‚Äî an E2 lockfile with no
  `links` key loads as an empty vec (`e2_lockfile_without_links_loads_...`).
- **Contract channel = E1.** The `contract` CLI (`vault-index`,
  `extract_contracts` + `diff_contracts`) reads the same normalized on-disk
  `.penpot` bytes the ledger hashes and classifies PATCH/MINOR/MAJOR. E3 pins
  the library contract hash into each `LibraryLink`; the bump surface is E1's.
- **Rebuild = M2.** A linked consumer is an ordinary vault file; a DB wipe
  resurrects both files by their original ids via the proven M2 in-place import.
  E3 adds only the re-derivation of the DB-only `file_library_rel` on top.

## The verbs (headless-triggerable, same-origin through the proxy)

- `POST /__api/packages/publish {id}` ‚Üí install (idempotent E2 path) +
  `set-file-shared`, record `libraryShared=true`. Returns `{fileId,
  libraryShared, deepLink}`. Publishing is **sticky** (E3 never auto-unpublishes
  ‚Äî un-publishing detaches every consumer server-side).
- `POST /__api/packages/link {consumerId, libraryId}` ‚Üí ensure the library is
  installed + published shared, ensure the consumer is installed, run the
  idempotent `link-file-to-library`, and pin a `LibraryLink` in the consumer's
  lock entry. Returns `{consumerFileId, libraryFileId, libraryRecursiveDeps}`.
- `GET /__api/packages` ‚Üí each package now carries `libraryShared:bool` +
  `links:[{libraryFileId, libraryPackageId, version, contractHash}]` ‚Äî the
  linked-state witness.
- Placing an instance is a direct backend RPC (`update-file`, one `add-obj`
  whose shape carries `{componentId, componentFile:<libId>, componentRoot,
  shapeRef}`, `skipValidate:true`) ‚Äî the proven probe sequence, no route needed.

## The model ‚Äî why the reference survives a DB wipe with no remap

The instance's `componentFile` is the **library's vault-local file-id** at
`data.pagesIndex.<page>.objects.<inst>.componentFile`. The one caveat the spike
found is the on-disk export flag: the daemon's default `embedAssets=true`
self-embeds and **severs** the link. So for a file with ‚â•1 lock link, the daemon
switches to `(includeLibraries=true, embedAssets=false)` and **trims the archive
to the file's own subtree** ‚Äî a one-file `.penpot` tree whose instance keeps
`componentFile=<libId>` as a bare id, with the library **not** inlined
(`include-libraries` as *storage* is the confirmed anti-pattern ‚Äî ¬ß7 of the
spike). Every unlinked file keeps the pre-E3 `embedAssets=true` path
byte-for-byte, so M2/M3/N5 cannot regress (the whole workspace suite stays
green). On a delete-DB rebuild, M2 resurrects both files by id, and the boot
re-link reconcile re-derives the DB-only `file_library_rel` from the lockfile
links once both endpoints are live ‚Äî no id remap (that is E6).

## The gate ‚Äî `scripts/e3-library.sh` (`just e3`)

Live stack, dedicated ports (proxy 8974, backend 6435, postgres 5509, valkey
6452, control 8975; exporter 6515 reserved, renders OFF). Fresh mktemp dirs,
ANSI-stripped log greps, pg-install cache seeded, dirs kept on failure,
run-twice idempotent. It authors a library package + a consumer package by
copying a materialized builtin-template tree into `.penpot-packages/`, publishes
+ links + places an instance, then asserts each exit criterion.

## Exit criteria (PLAN3 E3 ‚Äî all met)

- **(1) install+publish a library, author a consumer that links it and places an
  instance ‚Üí the instance's `componentFile` resolves to the library's
  vault-local id.** `/publish` returns `libraryShared:true`; the library's
  variant set + exported color + typography are authored into it; `/link` returns
  `consumerFileId` + `libraryFileId`; the placed instance reads back
  `componentFile == libId` in the DB **and** on disk. ‚úì
- **(2) delete-DB + reboot rebuilds BOTH files under their ORIGINAL ids and the
  instance STILL resolves (invariant 1), with `file_library_rel` re-derived from
  the lockfile.** After `rm -rf <data>/postgres` + reboot, `get-file` by id
  succeeds for both, the instance still carries `componentFile == libId`, and
  `lock.json` is byte-identical across the wipe. ‚úì
- **(3) a patch edit surfaces NO bump while minor/major surface the correct bump
  via the lockfile contract-diff channel, NOT `revn`.** An impl-only move reads
  `contract diff = PATCH` even though `revn` advanced (222‚Üí223); adding an
  exported color reads `MINOR`; removing one reads `MAJOR`. ‚úì
- **(4) `file_library_rel` is re-established from the lockfile on rebuild
  (derived/disposable).** Post-reboot, `get-file-libraries {consumer}` lists the
  library again with **no manual re-link** ‚Äî the bounded boot reconcile did it. ‚úì
- **(5) include-libraries is unused as storage.** The linked consumer's on-disk
  `.penpot` tree is single-file (`manifest.files == [consumer]`), `relations ==
  []`, the instance carries `componentFile=<libId>`, and there is **no**
  `files/<libId>‚Ä¶` subtree. ‚úì
- **(6) unlink clears the relation.** `unlink-file-from-library` ‚Üí subsequent
  `get-file-libraries` is empty. ‚úì

## Known limitation (E3 scope boundary ‚Äî documented, not a silent gap)

Spike ¬ß9 (`e3_flagcombo_probe.py`, `EXIT=0`) proved that **no single
`export-binfile` flag combination preserves both a library link and the
consumer file's own embedded raster media**: `(include=true, embed=true)` is
server-rejected, and the link-preserving `(include=true, embed=false)+trim`
detaches storage objects, so a linked consumer that ALSO uploads its own raster
media would lose that media on disk under E3's linked-file export. **E3's model
is a consumer that places component *instances* (vector, referenced by id),
which this does not affect.** The general "linked consumer with its own media"
case needs an upstream link-preserving + embed-own-media export mode (or a
two-export merge) and is deferred beyond E3. This is documented in the
`engine.rs` doc-comment and enforced by scope: the surgical path only applies to
files with ‚â•1 **lockfile** link.

## Notes / honest scope

- **`contract.rs` was NOT modified.** The library contract hash reuses E1's
  `extract_contracts` (already computed by `packages::contract_hash_of_tree`),
  pinned into `LibraryLink.contract_hash` at link time. E3 only **pins** it;
  turning the pinned hash into a live patch/minor/major *status surface* in the
  gallery is E4's job ‚Äî E3 proves the channel works via the `contract` CLI.
- **DB-only pointers preserved across re-install.** `do_install` now preserves
  `libraryShared` + `links` + `pluginProps` across a content re-import (they are
  DB-only pointers a re-import must not erase).
- **Editor-created links outside the package flow** (not in `lock.json`) are E4+
  scope ‚Äî only lockfile links drive the surgical export set and the boot reconcile.
