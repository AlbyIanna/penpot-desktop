# M1 — Process supervisor

## Goal (from PLAN.md)

Tauri app that launches embedded Postgres + Valkey + Penpot backend as supervised children,
provisions the single user, serves the frontend, opens one window auto-logged-in.
Clean shutdown kills children; crash of a child restarts it.

## Exit criteria (from PLAN.md)

`cargo tauri dev` gives a working offline Penpot with zero manual steps.

## Component checklist

- [x] **Fetch script** (`scripts/fetch-penpot.sh`): extracts the pinned Penpot 2.16.2 backend JAR +
      frontend static build (from the already-pulled Docker images) into `runtime/` (gitignored).
      Idempotent; jar-launch sanity check against the host JDK 26 on every run.
- [x] **Proxy** (`crates/proxy`): serves the frontend static build on 8686, proxies `/api` and the
      `/ws/notifications` WebSocket to the backend on 6161, implements X-Accel internal redirects
      for `/assets/` per `docs/m0/asset-serving.md`. 19 tests against a mock backend; the X-Accel
      path additionally verified against the live backend by the smoke test (step d).
- [x] **Supervisor** (`crates/supervisor`): launches embedded Postgres (5433), Valkey (6380), and
      the backend JVM (6161, `--enable-preview`, exact-major JDK 26) as children; readiness
      probes; restart with exponential backoff on crash; kills all on shutdown (no orphans —
      smoke step e); pinned `PENPOT_SECRET_KEY` persisted in the app-data dir.
- [x] **RPC client** (`crates/penpot-rpc`): login, single-user provisioning
      (`prepare-register-profile`/`register-profile` — shapes corrected against the live 2.16.2
      backend: `fullname` belongs to *prepare*, `register` takes only `token`), access tokens,
      binfile export/import (for M2). 23 wire-shape tests.
- [x] **Integration** (`apps/desktop`): shared boot sequence in `src/lib.rs` (config resolution
      with env overrides → secret-key persistence → supervisor → first-boot provisioning →
      proxy with `/__bootstrap` auto-login + rewritten `js/config.js`), used by both the Tauri
      shell (`src/main.rs`: placeholder page → navigate to `/__bootstrap` when ready, supervisor
      shutdown on exit) and the headless runner (`src/bin/headless.rs`, prints `READY <url>`).
      Backend is started via `clojure.main -e` instead of `-m app.main` because upstream's
      `-main` unconditionally opens an nrepl (remote code execution) listener on `0.0.0.0:6064`.
- [x] **Smoke test** (`just smoke` → `scripts/m1-smoke.sh`): fresh temp data dir; headless boot;
      asserts SPA through proxy, `/__bootstrap` Set-Cookie + redirect, authenticated get-profile
      with the persisted token, PNG upload → byte-identical fetch through `/assets/**` (X-Accel
      end-to-end), SIGTERM → clean exit with zero orphan java/valkey/postgres processes, and a
      second boot on the same data dir where the persisted token still authenticates
      (secret-key persistence). All steps PASS (2026-07-13).

## Notes / caveats

- Media uploads require ImageMagick's `identify` on `PATH` (backend shells out to it); installed
  via Homebrew on the dev machine. `node` is also required for SVG processing (present).
- The Tauri window flow (`cargo tauri dev`) compiles and uses the exact same boot path as the
  smoke-tested headless bin, but the GUI itself hasn't been driven by an automated test yet.
- Secret key + credentials live as mode-0600 files in the app-data dir; keychain storage is a
  TODO for M4.
- First boot needs the network once (embedded Postgres binary download); offline afterwards.

## Conventions

Ports (all defaults, all configurable; avoid 9001/6060 — used by the m0 spike stack):
proxy **8686**, backend **6161**, postgres **5433**, valkey **6380**.

Re-run from scratch by a separate verification agent (trust-nothing pass). Grades per
exit-criterion component:

- **`cargo test --workspace` green** — **MET**: 56 tests pass, 0 fail, from a full
  `cargo clean` rebuild (re-run warm: identical).
- **Launches Postgres + Valkey + backend as supervised children** — **MET**: during a live
  `cargo tauri dev` run, `ps` shows the app owning postgres (5433), valkey (6380) and the
  `--enable-preview` JVM; nothing listening on nrepl 6064; proxy bound to 127.0.0.1 only.
- **Provisions the single user** — **MET**: fresh data dir → mode-0600 `credentials.json`
  appears; `get-profile` with the stored token returns `local@penpot.test` (verified in GUI
  run and twice in smoke runs).
- **Serves the frontend through the custom proxy** — **MET**: `GET localhost:8686/` returns
  the real Penpot SPA; PNG upload → byte-identical fetch through `/assets/**` (smoke step d).
- **Opens one window auto-logged-in** — **MET (indirect)**: the window opened on this Mac and
  on *both* GUI boots an outside probe of `/__bootstrap` got `403 bootstrap already used this
  boot` — only the window can have consumed the one-shot login; the Set-Cookie relay itself
  was verified separately (`login-with-password` through the proxy → 200 + `auth-token`
  cookie). No pixel-level assertion (dev binary is not scriptable without Accessibility).
- **Crashed child restarts** — **MET (verified live, new evidence)**: with the stack running,
  `kill -9` valkey → respawned within seconds; `kill -9` the backend JVM → respawned and
  authenticated RPC recovered; a subsequent SIGTERM killed the *respawned* children too.
- **Clean shutdown kills children** — **MET** (after follow-up fix, 2026-07-13):
  - Headless SIGTERM: MET, verified in three smoke runs + once after crash-restarts.
  - GUI window-close (the intended user path): verified by construction only — same
    `RunningApp::shutdown` path as headless; not drivable non-interactively (no
    Accessibility grant; quit Apple event is ignored by the unbundled dev binary).
  - GUI killed by signal: originally NOT MET (tao installs no signal handler, so
    `RunEvent::Exit` never fired; SIGTERM orphaned all three children, Ctrl+C on
    `cargo tauri dev` orphaned postgres holding port 5433). **Fixed** in
    `apps/desktop/src/main.rs`: a SIGTERM/SIGINT handler routes through
    `handle.exit(0)` → `RunEvent::Exit` → supervisor shutdown. Re-verified live:
    (a) boot GUI → SIGTERM → app exits, zero orphans; (b) boot GUI in its own process
    group → group-wide SIGINT (faithful Ctrl+C simulation) → zero orphans, port 5433
    freed. `cargo test --workspace` still 56/56 green after the fix.
- **Exit criterion: `cargo tauri dev` = working offline Penpot, zero manual steps** —
  **MET** (boot and use verified from outside the app while it ran; shutdown gap closed
  by the fix above).

Flakiness note: smoke ran 3×. Runs 1 and 3 (isolated): ALL PASS. Run 2 failed only step (e)
because it ran concurrently with a second stack instance — the script's orphan check
(`pgrep -f penpot.jar`) was not instance-scoped and flagged the other instance's JVM.
Not an app bug; fixed by scoping the pgrep to children of the script's own headless pid
(`pgrep -P "$HEADLESS_PID"`).

### Implications for M2

- **Sync daemon hook point**: `boot()` in `apps/desktop/src/lib.rs` — after `provision()`
  it holds `backend_base_url` + the persisted access token (`credentials.json`), which is
  exactly the daemon's auth (`Authorization: Token …`). Start the daemon there, talking to
  the backend port directly (no proxy hop needed).
- **Backend crash window**: a killed backend JVM takes ~30–60 s to respawn and re-answer;
  the M2 daemon needs retry/backoff around `export-binfile`/`import-binfile`, and must treat
  RPC failure as transient, never as "file deleted".
- **Disposable-DB path exists**: `provision()` already re-registers on a wiped DB; M2's
  startup reconciliation (import everything on disk) should run right after that branch —
  that is the core-invariant restore path.
- **Fix GUI signal shutdown before the daemon lands**: an orphaned backend keeps accepting
  writes after the app is gone — with a sync daemon this becomes a data-consistency hazard,
  and the orphaned postgres blocks port 5433 for the next boot. Options: install a
  SIGTERM/SIGINT handler in the Tauri main that calls `app.exit()`, and/or a child watchdog
  (kqueue `EVFILT_PROC` on the parent; process-group kill), and startup reconciliation that
  detects and stops a stale postgres on our data dir before booting.
- **Smoke script**: scope the orphan check to `$DATA_DIR` (drop the global
  `pgrep -f penpot.jar`) so concurrent instances don't false-positive.
- Machine-level runtime deps to carry into M4 packaging: ImageMagick `identify`, `node`
  (backend shells out to both), JDK 26 exact major, valkey-server binary.
