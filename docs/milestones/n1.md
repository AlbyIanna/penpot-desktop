## Exit criteria (copied from PLAN2.md)

> `scripts/n1-index.sh` plants a needle string in one shape via `update-file`
> RPC, waits one sync window, queries `/__api/vault/search?q=` over HTTP and
> gets the correct file/page/board ids in <100ms against the torture fixture;
> deleting the index db + reboot rebuilds identical results from disk alone
> (invariant 1 proven); an edit renaming the needle removes the stale hit
> within the debounce window; run twice = no reindex churn (hash-gated
> no-op); `just e2e` exists and chains m1-smoke→m5→n1 green.

Reviewer note applied: the <100ms latency criterion is **soft-asserted**
(loud `WARNING` on miss) — correctness (exact ids) is the hard assert.

## What was built

- **`crates/vault-index`** — a self-contained sidecar cloned from the
  board-export recipe (`crates/board-export/src/{lib,state}.rs`):
  - Bundled-SQLite (rusqlite `bundled` — a vendored static lib, zero new
    host/runtime deps) **FTS5** index over the vault corpus: text-layer
    content, board (top-level frame) names, component names, library color
    names + hex values, typography names + font family
    (`src/extract.rs`; tree layout verified against the 2.16.2 backend's
    `app/binfile/v3.clj` writer and the real trees in `m0/roundtrip-work`).
  - Read-only `.penpot-sync.json` consumer: a dedicated thread polls the
    manifest every 1 s and reindexes a file **exactly when its
    `lastSyncedHash` moved** past the hash recorded in the db
    (`needs_reindex`, the `needs_render` pattern). Idle cycles are
    read-only — zero writes, watch-channel status published only on change.
  - **State lives inside the index db** (`indexed_files` table, written in
    the same transaction as the FTS rows) — deleting the db atomically
    deletes the state, which *is* chapter-2 invariant 1: the db is
    disposable, lives OUTSIDE the vault at
    `<data-dir>/vault-index/index.sqlite3`, and is never an input to sync.
  - **Sync-race handling (PLAN2.md risk 6):** the daemon saves the manifest
    *before* the directory swap lands (`crates/sync-daemon/src/engine.rs`
    — deliberate, so its watcher recognizes its own write). The indexer
    therefore records the semantic tree hash **of the bytes it actually
    read** (`sync_core::semantic_view` + `tree_hash` — the exact ledger
    semantics). A read that races the swap records the old hash, the next
    poll sees manifest ≠ recorded and reindexes: staleness self-heals and
    can never stick. Unit-tested (`mid_swap_read_self_heals`).
  - OS renames (manifest re-key, hash unchanged) update `rel_path` without
    a reindex; entries vanished from the manifest drop their rows.
  - Schema versioned via `PRAGMA user_version`; any mismatch discards and
    rebuilds (disposable cache).
- **Proxy surface** (same-origin via `Proxy::bind_with_router`, wired in
  `apps/desktop/src/lib.rs` — spawn + router merge + shutdown only):
  - `GET /__api/vault/search?q=&kind=&limit=` → JSON hits: kind, name,
    FTS5 snippet (`«match»` markers), fileId/pageId/objectId/boardId,
    relPath, bm25 score, and the **verified deep link**
    `/#/workspace?team-id&file-id&page-id` (route string in the compiled
    bundle; params destructured in upstream 2.16.2 `app/main/ui.cljs`).
    User input is never parsed as FTS syntax: every token is quoted
    (`query::build_match_query`), the last gets a `*` prefix marker for
    as-you-type search. Missing/empty `q` → 400; db not ready → 503.
  - `GET /__api/vault/status` → filesIndexed / filesPending / docsTotal /
    mutations / lastIndexAt / lastError (the gate's settle probe).
  - `GET /__search` → minimal server-rendered results page (plain
    HTML/vanilla JS, ~130 lines, no framework); clicking a hit is an
    ordinary same-origin navigation into the SPA. The lighttable comes in
    N3 — the value here is the query.
- **Shared seed tooling** (`scripts/n1_index_helper.py`, reused by N3/N5):
  torture-fixture seeder — `N1_FILES`×`N1_BOARDS` (default 100×10) files
  across 4 projects, one text layer per board with a deterministic unique
  token (`tokFFFxBB`, seeded RNG prose around it), plus a library color and
  typography per file; all via `create-file`/`update-file` RPC (payload
  shapes live-verified against 2.16.2: `growType` must be one of
  `auto-width|auto-height|fixed`; `add-color`/`add-typography` are flat
  camelCase maps; needle rename is `mod-obj` + `set content`).
- **`just e2e`** — the chapter-1 promise: chains
  m1-smoke → m2-invariant → m3-sync → m5-features → n1, serialized (the
  suites are concurrency-UNSAFE against sibling stacks — m4's lsof lesson).
  m4-artifact-test stays separate (needs a dmg build). `just n1` added.

## Gate blocks (scripts/n1-index.sh)

| Block | Asserts |
|---|---|
| (a) | 100 files / 1000 boards seeded via RPC, mirrored by the sync daemon, fully indexed; ≥ expected doc count |
| (b) | 23 deterministic queries (20 board tokens + color + typography + board name): exact file/page/board/shape ids + deep links (HARD); <100ms latency (SOFT, loud warning) |
| (c) | needle planted via `update-file` in a live board → correct hit within one sync window |
| (d) | unicode needle: CJK query matches; diacritic-folded query matches (`unicode61 remove_diacritics 2`) |
| (e) | idle poll cycles: mutation counter frozen (zero writes); reboot with db kept: ZERO reindexes (run-twice hash-gated no-op) |
| (f) | delete index db + WAL sidecars + reboot → full rebuild from disk alone; 9-query snapshot **byte-identical** (ranks + snippets included) |
| (g) | `mod-obj` rewrite of the needle text → stale hit gone within the window; new text searchable (rows swapped, not dropped) |
| (h) | `/__search` served same-origin; missing `q` → 400 |

Ports (ledger): proxy 8916, backend 6391, postgres 5465, valkey 6408.

## Observed numbers (full 100×10 run, 2026-07-14, this machine)

- fixture seed (201 RPC calls): **2.2 s**
- fixture fully mirrored to disk by the sync daemon AND indexed: **9 s**
  (2200 docs: 1000 boards + 1000 text layers + 100 colors + 100 typographies)
- query latency over the torture fixture (23-query battery through the
  proxy, urllib wall-clock): **p50 0.93 ms, mean 1.2 ms, max 6.85 ms** —
  the <100 ms criterion holds with ~15× headroom even at max
- needle end-to-end (`update-file` → correct hit over HTTP): **5.11 s**
  (dominated by the daemon's 2 s DB poll + export; the index adds ≤1 s);
  CJK needle 6.18 s
- reboot with db kept: 100 files recognized, **0 reindexes** (hash-gated)
- delete db + reboot: full rebuild from disk alone settled **10 s** after
  boot (100 reindexes), 9-query snapshot **byte-identical** to pre-deletion
- stale hit gone **5.65 s** after the rename edit; rewritten text
  searchable in the same cycle
- gate wall-clock: ~4 min with a warm pg-install cache

## Unit tests (cargo test -p vault-index — 20 tests)

- extraction from real normalized JSON shapes (the m0/roundtrip-work
  structures + upstream rich-text content tree), library assets, unicode,
  malformed-file tolerance;
- FTS query building: operator neutralization, quote escaping, prefix
  star, punctuation-only input — plus a live-SQLite proof that **no user
  input can make MATCH error**;
- `needs_reindex` decision table; deep-link building incl. defensive
  character filtering;
- db layer: replace/remove/re-key transactions, schema-version discard,
  read-only search on a missing db → NotReady;
- service integration: full cycle (index → no-op → edit → rename → delete),
  delete-db-rebuild-identical, mid-swap self-heal.

## Notes for the ladder

- The index deliberately also carries `kind=board` rows (board names) —
  N3's lighttable filter and N4's palette ranking read them for free.
- `SearchHandle` opens a read-only connection per query (µs-scale, WAL);
  no pooling needed at this corpus size (~22k docs << FTS5 comfort zone).
- The status endpoint's `mutations` counter is the churn oracle: any
  future regression that writes on idle cycles fails gate block (e).
- Components are covered by unit tests but not the live fixture (creating
  components via `update-file` needs component-root shapes; N6's template
  work will exercise that path live).
