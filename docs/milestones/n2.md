# N2 — Package the exporter as the render path

Status: **shipped** (2026-07-14). PLAN2.md milestone N2, risk 1 **DECIDED
option (a)** (product owner, 2026-07-14): pixel thumbnails are worth the dmg
growth — package the exporter (node + chromium headless shell) instead of the
wasm rasterizer / DB-resident canvas thumbnails / metadata-only cards.

## Exit criteria (copied from PLAN2.md)

> `scripts/n2-thumbs.sh` renders thumbnails for every board of a seeded vault
> on a **packaged-shape artifact** (no host node, no docker, offline — the
> chromium and node ship in the bundle and the m4 poisoned-proxy test still
> passes with renders on), second run a hash-gated no-op; degraded mode
> exercised in the same script; kill -9 during a failing render batch →
> restart adopts nothing stale and shutdown stays <20s (regression tests for
> both m5 bugs); `docs/milestones/n2.md` records the final dmg-size delta and
> per-board latency.

### 2. Bug fix A — stale-exporter adoption (post-M5 debt #1)

Boot could adopt a SIGKILL-orphaned exporter: the `/readyz` probe passed
against the stale process while our own child died of EADDRINUSE, and every
render then failed forever with the secret-key-mismatch 403. Fixed in three
layers (`crates/supervisor`):

- **Hard pre-boot refusal** (`lib.rs` step 4): before spawning, a connect
  probe on the exporter port + `lsof` pid lookup — busy port ⇒
  `SupervisorError::ServiceFailed` naming the pid(s):
  `exporter port <p> is already in use by pid(s) <pid> — refusing to adopt a
  stale exporter. Kill that process or set PENPOT_LOCAL_EXPORTER_PORT…`.
  A connect probe (not a bind test) because BSD `SO_REUSEADDR` lets a
  specific-address bind succeed while a stale wildcard `0.0.0.0` listener
  (the exporter's bind mode) still holds the port.
- **Per-(re)spawn guard + listener-identity check** (`child.rs`,
  `ChildSpec::listener_port`): every supervise-loop spawn re-checks the port,
  and readiness additionally verifies via `lsof` that the pid answering the
  probe IS our child (best-effort: no `lsof` ⇒ accept). A mismatch terminates
  the child and goes through the normal crash/GaveUp path.
- **Watchdog boot-window coverage**: the orphan-watchdog pid feeder now
  starts at the TOP of `Supervisor::start()` with a shared, growable slot
  list; every service registers its pid slot right after spawn (before
  readiness). In M5 the feeder only started after the whole boot succeeded,
  which is exactly how the exporter got orphaned. Residual exposure: the
  feeder's 1 s tick (documented; the n2 gate waits 1.2 s after the exporter
  binds before SIGKILLing).

### 3. Bug fix B — shutdown hang during failing renders (post-M5 debt #2)

SIGTERM during a failing render batch took 5–7 minutes: the per-board retry
ladder (~90 s of backoff) was not cancellation-aware and the run loop's
non-`biased` select let interval ticks outrace shutdown. Fixed in
`crates/board-export`:

- `retry::with_retry` now takes the shutdown `watch::Receiver` and races it —
  `biased`, shutdown branch first — against **both** every backoff sleep and
  the in-flight operation itself (`RetryError::Cancelled`);
- the run loop's select is `biased` with shutdown first; `poll_once` checks
  shutdown between due files; a cancelled render logs info (not error), arms
  no cooldown, and touches no disk;
- defense-in-depth: `ExporterClient` gained a 120 s reqwest timeout so a
  wedged (e.g. SIGSTOPped) exporter cannot hang a batch forever.

Regression tests: unit (`cancel_interrupts_a_retry_sleep`,
`cancel_interrupts_an_inflight_op`, `dropped_sender_counts_as_cancel`,
service-level `stop_during_a_failing_render_batch_is_prompt` under paused
time — fails on the old code, 20 s virtual budget) + live (n2 gate block (e):
SIGTERM during a real failing batch → **exit in 1 s**).

## Debts / notes for N3+

- Linux bundle variant unexecuted locally (CI-only, honest asymmetry as in
  M4); first `package.yml` run tells.
- **The exporter still binds 0.0.0.0** (upstream 2.16.2, listen host not
  configurable in the compiled bundle — M5 finding, unchanged by packaging).
  The render endpoint is cookie-authenticated, but the port is LAN-visible
  while `PENPOT_LOCAL_EXPORTS=1`. Candidate fixes for a hardening pass: a
  pf/socketfilterfw rule at install time, or a one-line upstream patch at
  bundle-build time (rejected for now — serve-time patching of upstream
  artifacts is against the project's grain).
- The stale-listener identity check is `lsof`-best-effort: environments
  without `lsof` (or with it off PATH — note it lives in `/usr/sbin` on
  macOS) fall back to the connect-probe guards only.
- Residual watchdog exposure: a SIGKILL landing within ~1 s of a child's
  spawn (one feeder tick) can still orphan that child; the pre-spawn port
  guard then refuses the next boot rather than adopting — fail-loud, not
  fail-wrong.
- Renders are serialized and cost ~5 s/board: a 100-file torture vault's
  FIRST render sweep is ~hours of background work. N3's lighttable must lean
  on the degraded placeholder + recency-ordered rendering; a
  priority/parallelism knob on board-export is candidate work if it hurts.
