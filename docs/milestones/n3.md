## What shipped

### 1. `/__api/vault/boards` ‚Äî the board listing (crate `vault-index`, read-only)

`crates/vault-index/src/boards.rs` + `db::all_boards`. Joins the index's
`kind = 'board'` rows with two disk sources the vault already owns:

- the sync **manifest** (`.penpot-sync.json`) ‚Üí project name + recency
  (`lastSyncedAt`) + the `.penpot` rel path;
- each file's **exports state** (`<name>.exports/.exports-state.json`, N2) ‚Üí
  the `board_id ‚Üí file_stem` map that names the rendered thumbnail file.

Each card carries the **exact** verified deep link
(`query::workspace_deep_link`, reused from N1) and a `thumb` URL when a render
exists, else `null` (N2 degraded mode). Query params: `project=` (filter,
still lists every project in `projects`) and `sort=recency|name` (recency is
newest-first; both have a deterministic tiebreak so a rebuilt index yields a
byte-identical listing ‚Äî invariant 1). Pure assembly (`assemble_cards`) is
unit-tested independent of disk; `all_boards` is a full scan of an UNINDEXED
FTS5 column, cheap at vault scale and never on the search hot path.

### 2. `/__home` + thumbnail + strip + reveal (crate `apps/desktop`)

`apps/desktop/src/home.rs` + `home.html`, merged into the proxy's extra router
(`Proxy::bind_with_router`) alongside N1's vault routes ‚Äî same-origin, plain
HTML/vanilla JS, no framework.

- **`GET /__home`** ‚Äî the grid page. Lazy-loaded `<img>` thumbnails with an
  inline SVG placeholder on `null`/load-error (degraded mode); project + sort
  controls; a name filter; a permanent **escape-hatch** anchor to
  `/#/dashboard/recent`. Cards are `<a href="/#/workspace?‚Ä¶">` ‚Äî a plain
  same-origin link click loads the SPA (works identically in the Tauri webview
  and the headless browser).
- **`GET /__api/vault/thumb?rel&board&fmt`** ‚Äî serves a board's N2 render from
  its `.exports` dir. The served filename (stem) comes from the **trusted**
  exports-state record, so the only client inputs are a within-vault `.penpot`
  path (traversal-checked), a board uuid, and `png`/`svg`. `404` = no render
  yet ‚Üí the page shows its placeholder.
- **`GET /__api/vault/strip`** ‚Äî the activity/conflict strip model (a poll
  endpoint; the page refreshes it every 1.2s ‚Äî chosen over SSE to stay
  dependency-free and trivially curl-assertable). Built purely from
  `SyncStatusSnapshot`; `Conflict{copy_path}` is first-class (rendered before
  everything, with the copy path surfaced).
- **`GET /__api/vault/reveal?path=`** ‚Äî reveal a `.penpot` dir or a
  `.conflict-‚Ä¶penpot` copy in the OS file manager, reusing the M5 reveal
  machinery (`reveal::reveal`). The path is constrained to a within-vault
  relative path (no `..`, no absolute) ‚Äî `400` on traversal.

### 3. `/__home` is the real landing view

`bootstrap_login`'s redirect target changed from `/` to **`/__home`**: every
boot now lands the webview on the lighttable with the auth cookie already set,
so card clicks deep-link straight into `/#/workspace?‚Ä¶` with the session
established. `main.rs` is unchanged (it already navigates to `/__bootstrap`).

### 4. The strip's status source (late-bound in `boot`)

The strip channel is created before the router (the router needs it) and its
feeder is spawned after the sync daemon:

- **production** ‚Üí mirrors the real `sync_daemon` `SyncStatusSnapshot` (so an
  edit's export/import/conflict flows onto the strip live);
- **CI / windowless** ‚Üí `PENPOT_LOCAL_TRAY_DEMO=1` feeds the strip
  `MockStatusSource::demo_frames()` (the same tray-demo source, now extended to
  the strip), so the Conflict + Error states and the reveal action are
  exercised deterministically with no real conflict to manufacture.

## Gates

- **`scripts/routes-gate.sh`** (NEW, PLAN2 risk 2 ‚Äî the SPA hash-route
  version-bump gate). Two legs: (STATIC) grep the route strings
  (`workspace`, `dashboard/recent`, `team-id`, `file-id`, `page-id`) out of
  the compiled `runtime/frontend/js/*.js`; (LIVE) a headless browser loads
  `/__home`, clicks one board card and asserts the landed URL is the exact
  `/#/workspace?team-id&file-id&page-id` deep link the card advertised, then
  triggers the escape hatch and asserts `/#/dashboard/recent`. Boots its own
  throwaway stack unless `ROUTES_GATE_BASE` points at a running one (n3-home
  passes its stack, so no double-boot). **Run this with `roundtrip.py` first
  after any Penpot version bump.**
  - **Browser/driver used:** the **bundled chromium headless shell** (N2's
    `runtime/exporter-browsers/`) driven by the **bundled playwright**
    (`runtime/exporter/node_modules/playwright`) via host `node` ‚Äî fully
    offline, no host Chrome. Asserts on navigation **commit** (`waitUntil:
    'commit'`) so the heavy ClojureScript SPA never has to finish booting.
    `scripts/routes_gate_nav.cjs` is the driver.
- **`scripts/n3-home.sh`** (NEW, `just n3`) ‚Äî N3's exit criteria verbatim,
  ports 8940/6401/5475/6418. Reuses the N1 torture-fixture seeder. Keeps
  renders OFF (the full 1000-board fixture stays fast) and proves the
  thumbnail path with a **planted** N2-shape render + degraded coexistence.
  The strip's conflict + reveal leg reboots the same stack with
  `PENPOT_LOCAL_TRAY_DEMO=1`.

## Invariants held

- **Invariant 1 (derived state disposable):** the listing reads only the index
  (rebuilt from disk), the manifest, and `.exports` ‚Äî nothing new is written;
  the sort/filter are deterministic so a rebuilt index lists identically.
- **Invariant 3 (SPA byte-untouched):** the only channel into the canvas is the
  deep-link URL; no injection, no plugin. `routes-gate.sh` is the standing
  guard on that one coupling.
- **Conflict rule (unchanged):** the strip is a *surface* for the M3 conflict
  state; it adds no new conflict semantics.
